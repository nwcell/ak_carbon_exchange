Description:
Abstract:
This data set contains ANCSA Selected and Conveyed lands within Alaska with respect to the surface estate for Interimly Conveyed (IC), Patented (PA), Conveyed (CV), or Selected (FR) village and regional native lands.
The Land Status attributes are a snapshot of the Land Information System Database and are only accurate as that database. The data is produced to the section (square mile) level. There is no dedicated land-to-village (or region) identifier in ALIS. What we do have is something called an "interest relationship" which identifies a case to a number of potentially interested parties. This retrieves a unique village or region most of the time, though not always. There are a few cases where an individual's name will be assigned. These are filtered out as we become aware of them.
See Process Steps for a list of interest relationship codes used. Only native case types are used. There may be other non-native owners in a given section, but they are not displayed.
Purpose:
This dataset was created for ANCSA conveyance identification for fire, and for general mapping purposes.

Logical_Consistency_Report:
Dataset based on query extract of the Alaska Lands Information System database (ALIS). The query used to extract Native Ownership information was based on village or regional corporation casetype codes and was limited to conveyed land selected ands (status codes: IC, PA, CV, FR);
Below is a description of the case types used for the extract: 265101 VILLAGE SELECTIONS-AK (Village, 12A ) 265102 REALLOCATED SELECTION-AK (Village, 12B ) 265200 REGIONAL SELECTIONS -AK (Regional, 12C ) 265202 IN-LIEU SELECTION - AK (Regional, 12A1) 265203 CIRI-NOM OUTSDE REG POOL (Regional, 12C ) 265204 CIRI-GSA POOL (Regional, 12C ) 265206 CIRI-FROM STATE THRU US (Regional, 12C ) 265208 CNI-12C/14H8 ANILCA EXCH (Regional, 12C ) 265209 CIRI-DFNCY RECON TO VLG (Regional, 12C ) 265210 CIRI-MISC & COAL/O&G (Regional, 12C ) 265302 NATIVE GROUP SEL-AK (Village, 14H2) 265303 FOUR CITIES SEL-AK (Village, 14H3) 265308 ANCSA 14(H)(8) SEL-AK (Regional, 14H8) 265400 NATIV RES-AK;ANCSA 19B&C (Village, 19BC)
The interest relationship codes used are: 01 APPLICANT, 30 PATENTEE, 63 ALSO KNOWN AS (AKA). For Village ownership, aka takes priority over applicant, which takes priority over patentee. For Regional ownership, only applicant is used.

Process_Step:
Process_Description:
Versions: SQL*Plus: Release 10.1.0.4.0 Oracle Database 10g Enterprise Edition Release 10.1.0.4.0 ARCINFO 9.2 ArcSDE 9.0 Oracle9i Build 1119
1. Truncate tables native_owner and native_area
2. From the NOTRUST database, insert Village and Regional data into the native_owner table.
3. For each unique occurrence of land status/casetype/status type/customer:
Export the sections related to the particular dissolve value to a coverage.
Clean coverage to .000001 tolerance.
Regiondissolve the separate sections into a single region.
Load the dissolved regions into the NATIVE_AREA layer.
5. Remove anomalous polygons.
6. Set the individual dissolve elements (see above) to their individual values.
